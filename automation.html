Accelerating hotel systems configurations with internal tooling

← Back

Context

At a global hospitality brand, multiple regional team members — including myself — were independently creating tools to speed up the configuration of hotel systems during migration to a new cloud property management system (PMS). These efforts were informal, uncoordinated and overlapping. We decided to consolidate and formalise our tooling development.

Process

As a Senior Associate with both technical skills and operational context, I took the lead on project structure, planning, and daily team coordination.

  1. Reviewed all existing scripts, tools and their functionality.
  2. Defined shared goals and minimum viable functionality.
  3. Set a 1-month target for a working demo and a 2-month target for full implementation.

I introduced GitHub for version control, trained the team on branching and collaboration practices, and set up GitHub Issues to track features and tasks. I ran daily stand-up and wrap-up calls to coordinate progress, distribute work, and maintain velocity of the project.

Outcomes

We delivered a working tool that automated pulling hotel configuration data from the legacy PMS and formatting it for use in our deployment dashboard.

•     Process time reduced from 90 minutes per property to 15 minutes

•     Successfully demoed early version within first month

•     Scaled tool to run across multiple properties simultaneously

•     Final version is being adopted internally, both in EMEA and US regions
